# 快捷指令 — gitpush & gitpull

- **创建时间**：2026-03-24 17:25
- **来源模块**：全局
- **优先级**：高
- **标签**：#Git #快捷指令 #每日必做

## 内容

### 📤 gitpush — 上传到 GitHub
**用途**：将本地所有变更提交并推送到 GitHub 远程仓库。

**操作步骤**：
1. 在项目根目录 `c:\AIforXuX` 下打开终端
2. 运行：`git add -A && git commit -m "提交信息" && git push`
3. 或直接双击 `gitpush.bat`（Windows 本地快捷脚本）

**提交信息格式**：`类型: 简短描述`
- `feat:` 新功能/新模块
- `docs:` 文档更新
- `add:` 添加内容/资料
- `fix:` 修复/修正
- `memory:` 记忆条目变更
- `refactor:` 结构重构
- `update:` 日常更新

**留空则自动生成**：`update: 日常更新 日期 时间`

---

### 📥 gitpull — 从 GitHub 拉取
**用途**：从 GitHub 远程仓库拉取最新内容到本地。

**操作步骤**：
1. 在项目根目录 `c:\AIforXuX` 下打开终端
2. 运行：`git pull origin main`
3. 或直接双击 `gitpull.bat`（Windows 本地快捷脚本）

---

### ⏰ 使用时机提醒

| 时机 | 操作 |
|------|------|
| **每天开始工作前** | 先执行 `gitpull` 拉取最新内容 |
| **每完成一个小任务** | 执行 `gitpush` 提交变更 |
| **每天下班前** | 必须执行 `gitpush` 确保远程备份 |
| **切换设备前** | 当前设备 `gitpush`，新设备 `gitpull` |
| **重大变更前** | 先 `gitpush` 保存当前状态 |
| **与 AI 协作结束后** | 执行 `gitpush` 保存会话产出 |

### ⚠️ AI 提醒规则
> **任何 AI 在协助用户完成一轮工作后，应主动提醒用户执行 `gitpush` 保存变更。**
> **任何 AI 在新会话开始时，应提醒用户先执行 `gitpull` 同步最新内容。**
