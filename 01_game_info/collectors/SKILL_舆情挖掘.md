
# 🗣️ SKILL — 社交媒体舆情挖掘 (Social Sentiment Miner)

> 自动从小红书抓取玩家评论，用 AI 分类分析，生成带设计建议的可视化报告。

**模块归属**：`01_game_info / collectors`
**当前状态**：✅ 已上线（v2.1）
**脚本文件**：`social_sentiment_miner.py`

---

## 🎯 功能定位

输入「游戏名 + 分析模块」，工具自动：

1. 在小红书搜索相关笔记（按最新排序，自动生成 4 个关键词组合）
2. 进入每篇笔记，抓取标题、正文、评论区内容
3. 用内置 UX 专项词库将内容归类到 **6 个体验维度**
4. 过滤攻略贴、广告、字数过少（< 8 字）等噪音
5. 生成 **HTML 报告**（4-Tab 结构）+ **Excel 数据表**

---

## 📐 分析维度（6 维度）

| 维度 | 关注点 | 负向例子 | 正向例子 |
|---|---|---|---|
| **易用性** | 操作是否顺畅、找得到 | "找半天找不到"、"步骤太多" | "一下就会"、"顺手" |
| **视觉认知** | 界面是否好看、清晰 | "花里胡哨"、"辣眼睛" | "颜值在线"、"清爽" |
| **系统反馈** | 加载/响应/稳定性 | "进不去"、"一直转圈" | "超流畅"、"不卡" |
| **趣味情感** | 好不好玩、有没有意思 | "坐牢感"、"想卸载" | "上头"、"停不下来" |
| **付费商业化** | 氪金体验是否合理 | "割韭菜"、"氪不起" | "良心游戏"、"不逼氪" |
| **联机社交** | 多人体验、匹配、社区 | "匹配不到人"、"喷子多" | "好友联机爽"、"合作愉快" |

## 🔑 UX 专项关键词库

### A. 操作与易用性
`操作反人类` `误触` `点不到` `步骤多` `返回键呢` `找半天` `卡住了` `引导太长` `强制教学` `哪里领` `断触` `路径深` `交互死循环`

### B. 视觉与认知负荷
`太乱了` `花里胡哨` `红点多` `字太小` `看不清` `遮挡` `屏效低` `没重点` `审美降级` `UI廉价` `光污染` `瞎了` `层级乱`

### C. 系统反馈与逻辑
`没反应` `提示不明显` `点了没用` `自动跳过` `动画太慢` `重连` `白点` `进度条` `确认框多` `卡死` `反馈弱` `加载久`

### D. 趣味性与情感化
`坐牢` `打工感` `没意思` `没成就感` `被恶心到` `仪式感` `惊喜` `细节好` `骗氪` `太肝了` `破防` `审美疲劳`

## 📋 调用接口

```python
# run_task.py 参数配置
game        = '我的世界'    # 游戏名称
module      = '山头服'      # 分析模块（如：新手引导/战斗系统/社交功能）
max_samples = 50            # 抓取笔记篇数（建议 20~50）
```

**工具接口 JSON Schema：**

```json
{
  "name": "social_sentiment_miner",
  "parameters": {
    "game_name":       { "type": "string",  "description": "游戏目标名称" },
    "target_module":   { "type": "string",  "description": "分析模块，如：周年庆UI、战斗系统" },
    "platforms":       { "type": "array",   "items": { "enum": ["xiaohongshu"] } },
    "analysis_depth":  { "type": "string",  "enum": ["quantitative", "qualitative", "full"] },
    "max_samples":     { "type": "integer", "default": 50 }
  }
}
```

## 🛡️ 反爬防御策略

| 机制 | 实现方式 |
|---|---|
| **会话持久化** | 加载本地 `cookies.json`，禁止游客模式高频请求 |
| **指纹随机化** | 随机切换 User-Agent，模拟 iOS/Android/Windows 特征 |
| **非线性延迟** | 请求间隔符合高斯分布（随机 3-12 秒），禁止等间距请求 |
| **交互模拟** | 模拟真实视口滚动 + 随机停顿，不直接请求 API |
| **熔断机制** | 连续 2 次触发验证码/403，立即停止并输出 `[STATUS] 触发反爬机制，任务熔断` |

## 📊 数据质量说明

| 质量分 | 含义 | 建议 |
|---|---|---|
| 85-100 | 数据可信度高 | 可直接用于决策 |
| 70-84 | 数据基本可用 | 建议人工复核几条原始数据 |
| < 70 | 噪音较多 | 需换关键词或扩大样本量 |

## ⚠️ 执行红线

1. **禁止自主创作**：有效评论 < 10 条时，必须如实报告样本不足，不得强行分析
2. **隐私脱敏**：引用玩家评论时必须隐去 ID 及个人敏感信息
3. **证据闭环**：每项诊断结论必须有真实社交媒体数据作为支撑
4. **降级告知**：因反爬导致样本量低于预期时，必须在报告开头显著声明数据局限性
5. **Cookie 安全**：`cookies.json` 严禁共享或上传公开平台，有效期约 30 天需定期更新

## 🍪 Cookie 配置步骤

1. Chrome 登录小红书（www.xiaohongshu.com）
2. `F12` → Application → Cookies → `www.xiaohongshu.com`
3. 提取 `a1` / `web_session` / `webId` 三个关键字段
4. 保存至 `collectors/cookies.json`：

```json
{
  "xiaohongshu": [
    {"name": "a1",          "value": "你的值", "domain": ".xiaohongshu.com"},
    {"name": "web_session", "value": "你的值", "domain": ".xiaohongshu.com"},
    {"name": "webId",       "value": "你的值", "domain": ".xiaohongshu.com"}
  ]
}
```
