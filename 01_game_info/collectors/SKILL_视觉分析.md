
# 📸 SKILL — 静态界面视觉诊断 (Image-UX Analyzer)

> 对游戏截图进行专业级 UX 视觉分析，量化可用性问题，输出有设计理论支撑的诊断报告。

**模块归属**：`01_game_info / collectors`
**当前状态**：🔧 开发中（Phase 1 待排期）
**脚本文件**：`vision_engine.py`

---

## 🎯 功能定位

输入一张或多张游戏界面截图，自动输出：

1. **视觉流线分析** — 模拟玩家前 3 秒视线路径，判断核心功能是否处于视觉热区
2. **控件热区检测** — 测量按钮/图标像素大小，验证是否满足触控标准（≥44dp）
3. **信息层级诊断** — 检测同屏元素数量，识别信息过载区域
4. **情感色彩评估** — 评估界面配色是否与游戏品类氛围匹配

---

## 📐 分析维度

| 维度 | 理论依据 | 具体检测内容 |
|---|---|---|
| **视觉流线** | 显著性检测（Saliency Map）| 前3秒视线焦点转移路径，核心入口是否可见 |
| **控件热区** | 费茨定律（Fitts's Law）| 按钮尺寸是否 ≥44dp，热区是否存在误触风险 |
| **信息层级** | 认知负荷理论（Cognitive Load）| 同屏元素数量，是否存在视觉噪声遮盖关键信息 |
| **情感美学** | 色彩心理学 + 格式塔原则 | 配色与品类匹配度，界面氛围感评分 |

## 📊 输出规范示例

```
[界面诊断报告] ui_battle_001.jpg

[视觉流线]
  ✅ 核心按钮「开始战斗」位于视觉热区（右下1/3区域），符合拇指操作习惯
  ⚠️ 右上角「活动」入口与「设置」图标间距仅 8px，存在误触风险

[控件热区 - Fitts's Law]
  ❌ 左上角「返回」按钮实际热区仅 32×32px，低于移动端建议值 44dp
  ✅ 主操作区所有按钮热区 ≥ 48dp，符合规范

[信息层级 - 认知负荷]
  ⚠️ 同屏可见元素达 23 个，超出建议上限（7±2），存在注意力分散风险
  ❌ 「限时活动」浮层遮盖了「日常任务」入口

[情感美学]
  ✅ 界面配色（深蓝 + 金色）与竞技品类氛围高度匹配

[开发成本评估]
  控件热区修复 → L（资源替换）
  信息层级优化 → M（布局调整）
```

## 🛠️ 技术方案

| 组件 | 技术选型 | 用途 |
|---|---|---|
| 显著性检测 | OpenCV Saliency / MediaPipe | 生成视觉热力图 |
| 控件识别 | GPT-4o Vision / Gemini Pro Vision | 自动标注 UI 元素 |
| 尺寸测量 | PIL / OpenCV | 像素级控件测量 |
| 输出报告 | HTML + Canvas 标注图 | 可视化展示诊断结果 |

## 📋 调用接口（规划）

```python
# 调用示例
from collectors.vision_engine import analyze_ui

result = analyze_ui(
    image_path = "resources/screenshots/ui_battle_001.jpg",
    focus_areas = ["战斗UI", "主菜单", "抽卡演出"],
    theory_standard = ["Fitts", "Gestalt", "Nielsen"]
)
```

**参数说明：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `image_path` | string | 截图本地路径或 URL |
| `focus_areas` | array | 重点分析模块列表 |
| `theory_standard` | array | 参考 UX 理论标准 |

## ⚠️ 执行红线

1. **禁止自主创作**：所有结论必须挂载 `[截图编号]` 作为证据，严禁脑补视觉流线
2. **开发落地性**：建议必须标注是「UI 资源优化（低成本）」还是「交互逻辑重构（高成本）」
3. **数据溯源**：每项热区诊断结论需附上像素测量数据
