
# 🎬 SKILL — 游玩视频时序解析 (Video-Flow Analyzer)

> 从游戏录屏中自动识别玩家卡点、关键操作节点，还原真实的操作路径与体验流。

**模块归属**：`01_game_info / collectors`
**当前状态**：✅ Phase 1 已完成，Phase 2/3 待排期
**脚本文件**：`video_parser.py`

---

## 🎯 功能定位

输入一段游戏录屏，自动输出：

1. **场景切割** — 自动识别界面切换节点（如：主界面 → 战斗 → 结算）
2. **卡点捕捉** — 找出玩家长时间停顿、犹豫、重复操作的时刻
3. **关键事件标记** — OCR 识别「失败」「领取」「充值」等高价值画面
4. **行为熵分析** — 检测无效操作循环（如：反复开关同一界面）

---

## 🔍 三态触发机制（核心原理）

拒绝机械等分帧，仅抓取具备「分析价值」的瞬间：

| 触发类型 | 触发条件 | 捕捉目标 |
|---|---|---|
| **突变触发** *(Delta Trigger)* | 画面变化率 > 30%（直方图/特征点）| 界面切换、奖励获得、场景转场 |
| **停顿触发** *(Stall Trigger)* | 画面中心位移矢量连续 3s 趋近于 0 | 玩家卡点、迷路、阅读困难引导 |
| **标记触发** *(OCR Trigger)* | OCR 识别到「失败/领取/充值/恭喜获得」| 高价值转化路径、情绪爆发点 |

## 🛠️ 技术方案

| 组件 | 技术选型 | 用途 |
|---|---|---|
| 场景分割 | PySceneDetect | 自动切割场景边界 |
| 停顿检测 | Optical Flow（光流算法）| 识别玩家卡点 |
| 文字识别 | PaddleOCR / Tesseract | 关键事件标记 |
| 视频处理 | FFmpeg + NumPy | 帧提取与处理 |
| 输出报告 | HTML + 缩略图时间轴 | 可视化展示分析结果 |

## 📋 调用接口（规划）

```python
# 调用示例
from collectors.video_parser import analyze_video

result = analyze_video(
    video_path      = "resources/videos/gameplay_01.mp4",
    stall_threshold = 3.0,                                         # 判定卡点的停顿秒数
    custom_keywords = ["失败", "领取", "充值", "恭喜获得"]          # OCR 重点监控词
)
```

**参数说明：**

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `video_path` | string | — | 视频文件路径 |
| `stall_threshold` | float | `3.0` | 判定为卡点的停顿秒数 |
| `custom_keywords` | array | `["失败","领取","充值"]` | OCR 需重点监控的关键词 |

## ⚠️ 执行红线

1. **禁止自主创作**：所有卡点结论必须挂载 `[视频秒数]` 作为证据，严禁脑补玩家行为
2. **开发落地性**：建议必须标注是「UI 资源优化（低成本）」还是「交互逻辑重构（高成本）」
3. **排除剧情动画**：停顿触发需排除剧情 CG 播放段落，避免误判为卡点
4. **数据溯源**：行为熵分析中每条无效循环需附具体时间戳区间
