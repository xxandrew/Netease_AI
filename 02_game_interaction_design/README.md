# 🎨 02_game_interaction_design — 游戏交互设计

> 负责游戏交互设计相关的原型、UI/UX 模式、流程图和参考资料管理。

---

## 模块概述

本模块聚焦游戏交互设计领域，辅助设计师进行用户体验设计、界面原型搭建、交互流程梳理等工作。涵盖从设计研究到规范沉淀的完整工作链路。

---

## 子目录说明

| 目录 | 说明 |
|------|------|
| `prototypes/` | 原型设计文件 — 线框图、交互原型、设计稿 |
| `ui_patterns/` | UI/UX 模式库 — 常用交互模式、设计规范、组件库 |
| `flowcharts/` | 交互流程图 — 用户旅程、操作流程、状态机 |
| `references/` | 参考案例 — 优秀设计案例、竞品交互分析 |

---

## 🛠️ 关联技能包（Skills）

本模块与 `01_game_info` 的分析技能深度联动，以下技能的输出可直接驱动交互设计决策：

| 技能 | 来源模块 | 与本模块的关系 |
|------|---------|--------------|
| 🕹️ **界面易用性分析** | `01_game_info` | 分析结果 → 驱动 `ui_patterns/` 规范更新 |
| 📸 **视觉诊断** | `01_game_info` | 视觉流线分析 → 指导 `prototypes/` 布局决策 |
| 🗣️ **舆情挖掘** | `01_game_info` | 玩家痛点 → 转化为 `flowcharts/` 中的流程优化点 |
| 🧠 **系统分析** | `07_general` | 方案评估 → 支持设计方向取舍决策 |

---

## 🔄 工作流程

```
需求输入
    ↓
references（参考研究 + 竞品分析）
    ↓
flowcharts（用户旅程 + 交互流程设计）
    ↓
prototypes（线框图 + 交互原型制作）
    ↓
ui_patterns（规范沉淀 + 组件复用）
    ↑
[迭代反馈] ← 易用性分析 / 舆情数据
```

---

## 📋 快速入口

- **要做流程梳理** → 在 `flowcharts/` 创建用户旅程文件
- **要做交互原型** → 在 `prototypes/` 创建原型文件，参考 `ui_patterns/` 规范
- **要找参考案例** → 查看 `references/` 竞品分析归档
- **要分析现有界面** → 联动 `01_game_info/collectors/SKILL_界面易用性分析.md`

---

## 📌 设计文件规范

- 原型文件命名：`YYYYMMDD_功能名称_v版本号`（如 `20260401_战斗HUD_v1.0`）
- 流程图需同时覆盖**正常流程**和**异常/边界流程**
- 参考案例需标注：来源游戏名 + 分析要点 + 适用结论
- 设计稿需注明目标平台（移动端 / PC / 主机）
