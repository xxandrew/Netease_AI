# 🔄 PPT 协作制作工作流 (SOP)

> **核心理念**：用户提供方向和判断，AI 负责结构化和内容生产。  
> **工作模式**：渐进式协作 — 先确认骨架，再填充血肉，最后精修打磨。

---

## 📌 工作流总览

```
用户：说主题 + 目标（1-2句话）
        ↓
AI：提出补充问题（明确受众、时长、重点）
        ↓
AI：输出【第一版大纲】（只有页面标题+一句话说明）
        ↓
用户：审核大纲 ✅确认 / ✏️调整
        ↓
AI：输出【每页详细内容】（要点+数据框+图表建议）
        ↓
用户：逐页审核 ✅通过 / ✏️修改具体页面
        ↓
AI：输出【最终版 + 讲稿备注（可选）】
        ↓
用户：复制到 PowerPoint / WPS 完成制作
```

---

## 🚀 第一步：启动 PPT 制作

### 用户输入格式（越简单越好）

```
我要做一个PPT：
主题：[一句话说清楚]
目标：[这个PPT要达到什么效果，如：让领导批准立项/汇报本周进展/分析某款竞品]
受众：[谁来看，如：部门总监/项目全体成员]
时间：[汇报时长，如：10分钟/不限]
其他补充：[有就填，没有就不填]
```

**示例：**
```
我要做一个PPT：
主题：为一款卡牌策略手游申请立项
目标：让产品总监批准进入预研阶段
受众：产品总监 + 技术总监
时间：15分钟
其他补充：竞品参考《XX》，核心差异化是联机对战玩法
```

---

## 🤖 第二步：AI 输出第一版大纲

**AI 的输出标准：**
- 只给页面标题 + 一句话说明（不展开内容）
- 给出建议页数和理由
- 标注哪些页是重点（需要重点打磨）
- 询问用户是否有调整

**示例输出格式：**

```
📋 PPT大纲 v1.0 — 卡牌策略手游立项申请（共12页，建议15分钟）

第01页｜封面          → 项目名称 + 立项申请字样
第02页｜一句话定位    → [★重点] 用最简洁的方式说清楚这是什么游戏
第03页｜市场机会      → 为什么现在做这个品类
第04页｜产品定位      → 我们做什么、不做什么
第05页｜核心玩法      → [★重点] 玩法核心循环拆解
第06页｜差异化竞争力  → [★重点] 和竞品的本质区别
第07页｜目标用户      → 谁会玩这个游戏
第08页｜商业化策略    → 如何盈利
第09页｜开发计划      → 里程碑和时间节点
第10页｜资源需求      → 需要多少人/钱/时间
第11页｜风险与应对    → 主要风险和预案
第12页｜总结与诉求    → 明确提出需要批准什么

---
❓ 请确认大纲结构：
① 整体方向是否正确？
② 哪些页面需要调整/增加/删除？
③ 有没有特别想强调的部分？
```

---

## ✏️ 第三步：用户审核大纲

用户可以这样回复：

```
✅ 整体OK，但：
- 第07页（目标用户）不需要，删掉
- 在第05页后面加一页：核心玩法视频/原型演示说明
- 第12页的诉求要更突出，单独做重点页
```

或者：

```
✅ 全部OK，按这个大纲继续细化
```

---

## 📝 第四步：AI 输出每页详细内容

**AI 的输出标准：**
- 逐页展开，格式统一
- 每页包含：标题 / 核心要点（3-5条）/ 图表建议 / 备注说明
- 重点页面给出两个方案供选择

**示例输出格式：**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第05页｜核心玩法  [★重点页]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【页面标题】：玩家在游戏里做什么？

【核心内容】：
▸ 核心循环（30秒）：抽卡 → 组建卡组 → 对战 → 获得奖励 → 强化卡牌
▸ 核心乐趣点①：组合策略——数百种卡牌组合，每次对战都有新解法
▸ 核心乐趣点②：实时联机——与真实玩家1v1对战，而非AI
▸ 差异化：市面上同类产品均为异步对战，我们是实时对战
▸ 学习曲线：3分钟上手，深度玩法需要数十小时探索

【图表建议】：核心循环流程示意图（环形箭头图）+ 对战截图/原型图

【讲稿备注】：这页停留约2分钟，重点强调"实时联机"是与竞品的核心区别

---
⚠️ 方案B（如果想更简洁）：
直接用一张核心循环图 + 3个关键词（策略/联机/深度），配口头说明
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## 🔁 第五步：用户逐页审核 & 局部修改

用户可以这样回复：

```
整体好，修改意见：
- 第05页：差异化那条改成"全球首款实时联机卡牌策略手游（移动端）"，更有力
- 第09页（开发计划）：时间节点太乐观，把Alpha时间从M6改到M8
- 第11页：风险页面要加一条"核心玩法留存率风险"
- 其余页面 ✅ 通过
```

AI 只修改被指出的页面，不动其他内容。

---

## 🎙️ 第六步（可选）：生成讲稿备注

用户可以说：

```
帮我生成完整讲稿，总时长15分钟，风格正式但不刻板
```

AI 按每页时间分配生成配套讲稿。

---

## 📌 AI 在此工作流中的行为规则

> **AI 必须遵守以下规则：**
>
> 1. **不跳步** — 大纲没确认前，不主动展开详细内容
> 2. **不假设** — 用户信息不足时，主动提问，不自行补全关键信息
> 3. **最小修改原则** — 用户只说改某页，AI 只改那页，不动其他页
> 4. **给选项而非替换** — 有更好方案时，以"方案B"形式提供，不直接覆盖
> 5. **版本标记** — 每次输出标注版本号（v1.0大纲 / v1.1大纲 / v1.0内容...）
> 6. **主动推进** — 每次输出结束后，明确告诉用户下一步需要做什么
