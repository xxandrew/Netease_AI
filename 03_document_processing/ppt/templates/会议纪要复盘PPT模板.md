# 📝 会议纪要 / 复盘 PPT 模板

> **适用场景**：会后复盘汇报、项目阶段复盘、事故/问题复盘、例会纪要整理
> **建议页数**：6 - 10 页
> **汇报时长**：10 - 20 分钟

---

## 📋 PPT 完整结构模板 A：会议纪要

---

### 第1页｜封面
```
会议纪要
会议主题：[主题]
时间：XXXX年XX月XX日 XX:XX — XX:XX
地点：[线上/线下 + 地点]
记录人：XXX
```

---

### 第2页｜会议概况
```
标题：会议概况

▸ 参会人员：[姓名/部门列表]
▸ 会议背景：[为什么召开这次会议，1-2句]
▸ 本次议题：
  1. [议题1]
  2. [议题2]
  3. [议题3]
```

---

### 第3页｜核心决策事项
```
标题：本次会议决定了什么？

内容框架（表格）：

| 决策事项 | 决策结论 | 决策依据 |
|---------|---------|---------|
| 事项1 | [结论] | [依据] |
| 事项2 | [结论] | [依据] |
| 事项3 | [结论] | [依据] |

重要原则：决策即行动，此处内容应清晰无歧义。
```

---

### 第4页｜待办事项（Action Items）
```
标题：接下来谁做什么？

内容框架（表格）：

| 待办事项 | 负责人 | 协作人 | 截止时间 | 优先级 |
|---------|--------|--------|---------|--------|
| 任务1 | XXX | XXX | MM/DD | 高 |
| 任务2 | XXX | — | MM/DD | 中 |
| 任务3 | XXX | XXX | MM/DD | 高 |

备注：[需要额外说明的内容]
```

---

### 第5页｜遗留问题
```
标题：本次未解决的问题

内容框架：

▸ 遗留问题1：[描述] 
  → 下一步：[谁来跟进 / 何时再议]
  
▸ 遗留问题2：[描述]
  → 下一步：[谁来跟进 / 何时再议]

下次会议议题预排：
  1. [议题1]
  2. [议题2]
```

```

### 第3页｜目标 vs 结果对比
```

---

### 第4页｜做得好的地方（亮点）
```
标题：哪些地方做得好？为什么好？

内容框架（每点用 STAR 结构描述）：

✅ 亮点1：[标题]
  情况：[背景]
  我们的做法：[具体行动]
  结果：[产生了什么正面效果]
  可复用的经验：[下次可继续沿用的方法]

✅ 亮点2：（同上）

✅ 亮点3：（同上）
```

---

### 第5页｜做得不好的地方（问题）
```
标题：哪些地方做得不好？根本原因是什么？

内容框架：

```

## 💡 制作技巧

| 技巧 | 说明 |
|------|------|
| **对事不对人** | 复盘聚焦问题和流程，不是追责，语言要中性 |
| **根本原因优先** | 找到根本原因比描述现象更重要，用5Why追问 |
| **Action Items必须有人有时间** | 每个改进措施必须指定负责人+截止时间 |
| **会议纪要24小时内发出** | 会议结束后尽快整理发给与会者确认 |
| **亮点也要复盘** | 不只复盘失败，成功经验同样要总结可复用点 |
