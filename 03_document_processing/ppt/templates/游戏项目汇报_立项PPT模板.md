# 🎮 游戏项目汇报 / 立项PPT 模板

> **适用场景**：新项目立项汇报、阶段性项目进展汇报、项目评审会
> **建议页数**：12 - 18 页
> **汇报时长**：15 - 30 分钟

---

```

## 📋 PPT 完整结构模板

---

### 第1页｜封面
```
【项目名称】
副标题：项目汇报 / 立项申请
汇报人：XXX   部门：XXX   日期：XXXX年XX月XX日
```

---

### 第2页｜目录/议程
```
本次汇报内容
1. 项目背景与机会
2. 产品定位与核心玩法
3. 目标用户分析
4. 竞品分析
5. 核心竞争力
6. 开发计划与里程碑
7. 团队与资源需求
8. 风险与应对
9. 总结与诉求
```

---

### 第3页｜项目背景 & 市场机会
```
标题：为什么做这个游戏？

内容框架：
▸ 市场现状：[当前市场规模/趋势数据]
▸ 用户痛点：[目标用户目前缺少什么]
▸ 市场空白：[竞品未满足的需求点]
▸ 时机判断：[为什么是现在]

参考数据来源：App Annie / 伽马数据 / 公司内部数据
```

---

### 第4页｜产品定位
```
标题：这是一款什么样的游戏？

内容框架：
▸ 一句话定位：[类比句式，如：像X游戏一样，但拥有Y特点]
▸ 游戏类型：[细分品类]
▸ 目标平台：[iOS / Android / PC / 主机]
▸ 商业模式：[买断 / 内购 / 混合变现]
▸ 预期用户规模：[DAU目标]

可加入：竞品定位象限图
```

---

### 第5页｜核心玩法
```
标题：玩家在游戏里做什么？

内容框架：
▸ 核心循环（Core Loop）：[30秒/3分钟/30分钟循环拆解]
▸ 核心乐趣点：[3个关键爽点]
▸ 差异化玩法：[与竞品的本质区别]

建议配图：核心循环示意图 / 玩法流程图
```

---

### 第6页｜目标用户分析
```
标题：谁会玩这个游戏？

内容框架：
▸ 核心用户画像：[年龄/性别/游戏习惯]
▸ 用户需求：[玩家核心诉求Top3]
▸ 用户行为特征：[何时玩/玩多久/付费习惯]
▸ 用户规模估算：[TAM/SAM]

建议配图：用户画像卡片
```

|------|-------|-------|-----------|

| 目标用户 | | | |

| 优势 | | | |
| 劣势 | | | |

```

---

### 第8页｜核心竞争力
```
标题：我们凭什么赢？

内容框架：
▸ 差异化优势1：[具体描述]

### 第9页｜商业化策略
```

内容框架：

▸ 预期LTV：[单用户生命周期价值估算]

---

### 第10页｜开发计划 & 里程碑
```
标题：怎么做？做多久？

内容框架（甘特图或时间轴）：

| 阶段 | 时间 | 目标 | 交付物 |
|------|------|------|--------|
| 概念验证 | M1-M2 | 核心玩法可玩 | Demo版本 |

| Beta | M9-M12 | 全功能+优化 | 测试版 |
| 上线 | M13 | 正式发行 | 正式版 |
```

### 第13页｜总结 & 诉求

## 💡 制作技巧

| 技巧 | 说明 |
|------|------|
| **数据优先** | 每页尽量有1个数据支撑，避免纯文字 |
| **一页一重点** | 每页只传达1个核心信息 |
| **结论前置** | 先给结论，再给依据（管理层看PPT习惯） |
| **图大字少** | 领导层汇报：字越少越好，配图越大越好 |
| **3-5条原则** | 每页要点控制在3-5条，不超过5条 |
