
# 🧠 SKILL — 系统分析 (Systematic Analysis)

> 辩证拆解任何问题，从多维度推演可能性，用结构化输出驱动清晰决策。

**模块归属**：`07_general`
**当前状态**：✅ 核心框架已定义
**适用场景**：战略决策 / 设计方案评估 / 问题归因 / 机会判断

---

## 🎯 核心能力定位

接收任何「问题/决策/方案」作为输入，系统性输出：

1. **核心结论前置** — 先给答案，再展开推导
2. **辩证拆解** — 正面/负面/中间态三向分析
3. **多维度覆盖** — 按预设分析框架逐层展开
4. **可能性推演** — 列举各路径结果及概率权重
5. **价值与影响评估** — 量化或定性评估每个结论的影响半径
6. **可视化辅助** — 必要时用流程图/矩阵/对比表增强表达

## 📐 标准分析框架

### 第一层：问题解构（是什么）

```
问题
├── 表层现象    ← 能直接观测到的
├── 根因假设    ← 推测的底层原因（需标注「假设」）
└── 问题边界    ← 这个问题的影响范围到哪里为止
```

### 第二层：辩证分析（正 / 反 / 中）

| 视角 | 内容 | 权重参考 |
|---|---|---|
| ✅ **支持面** | 这个方向成立的证据和逻辑 | 高/中/低 |
| ❌ **反对面** | 这个方向不成立的证据和逻辑 | 高/中/低 |
| ⚖️ **中间态** | 条件成立时才有效的灰色地带 | — |

### 第三层：路径推演（如果→那么）

```
当前状态
    ├── 路径 A（最可能）→ 结果A → 影响A
    ├── 路径 B（次可能）→ 结果B → 影响B
    └── 路径 C（极端情况）→ 结果C → 影响C
```

### 第四层：价值与影响评估

| 维度 | 说明 | 评估方式 |
|---|---|---|
| **短期影响** | 0-3个月内能感知到的变化 | 定性描述 |
| **长期价值** | 6个月以上的战略价值 | 定性+量级估算 |
| **副作用** | 可能带来的负面连锁反应 | 风险等级（H/M/L）|
| **机会成本** | 选择这条路放弃了什么 | 对比描述 |

## 🗺️ 常用可视化图表类型

| 场景 | 推荐图表 | 说明 |
|---|---|---|
| 因果关系梳理 | **鱼骨图 / 因果链** | 找根因，追溯链条 |
| 多方案对比 | **决策矩阵表** | 多维度打分横向对比 |
| 流程/路径推演 | **流程图** | 分支路径+结果节点 |
| 优先级判断 | **2×2 矩阵**（重要×紧急）| 快速筛选行动项 |
| 影响范围评估 | **影响力地图** | 利益相关方受影响程度 |
| 可能性分布 | **决策树** | 多层级路径概率推演 |

### 示例：2×2 优先级矩阵

```
        重要
         ↑
  [立刻做] │ [计划做]
  ─────────┼─────────  → 紧急
  [可以删] │ [授权做]
```

### 示例：因果链

```
[根因] → [触发条件] → [中间状态] → [表层现象] → [最终影响]
  ↑                                                    │
  └──────────────── 反馈回路 ──────────────────────────┘
```

## 🔄 分析流程 SOP

```
接收问题
    ↓
① 快速判断：这是「事实型」还是「判断型」问题？
    ├── 事实型（有标准答案）→ 直接查证/推导，给出确定结论
    └── 判断型（无标准答案）→ 进入完整分析框架
            ↓
        ② 问题解构（明确范围和根因）
            ↓
        ③ 辩证分析（正反中三向）
            ↓
        ④ 路径推演（列出可能结果）
            ↓
        ⑤ 价值与影响评估
            ↓
        ⑥ 结论前置输出（核心结论 → 展开说明）
            ↓
        ⑦ 判断是否需要可视化辅助
```

## ⚙️ 输出行为规则

| 规则 | 说明 |
|---|---|
| **结论前置** | 每次分析必须先给核心结论，再展开论证 |
| **标注假设** | 凡是推测性内容，必须标注「[假设]」或「[待验证]」 |
| **层级清晰** | 用分隔线区分不同分析层级，禁止混写 |
| **证据挂载** | 每个判断需说明依据（数据/案例/逻辑推导），不凭空下结论 |
| **量力而行** | 信息不足时，说明缺口在哪，不强行得出完整结论 |
| **可视化按需** | 文字能说清楚的不强行画图；关系复杂时主动使用图表 |
| **禁止空谈** | 所有分析结论必须指向「所以应该怎么做」，不做无行动建议的纯理论推演 |

## 💡 典型调用场景

| 场景 | 示例输入 | 分析侧重 |
|---|---|---|
| **设计方案评估** | 「这个交互方案值得做吗？」 | 辩证分析 + 机会成本 |
| **问题归因** | 「为什么留存率下降了？」 | 问题解构 + 因果链 |
| **机会判断** | 「要不要做这个功能？」 | 路径推演 + 价值评估 |
| **战略取舍** | 「A 和 B 方向选哪个？」 | 决策矩阵 + 影响评估 |
| **风险预判** | 「这样做会有什么问题？」 | 副作用分析 + 极端路径 |

## ⚠️ 执行红线

1. **禁止自主创作数据**：引用数据必须来自真实来源，没有数据时用「[数据缺口]」标注
2. **禁止空谈理论**：分析必须结合具体场景，不做脱离实际的哲学推演
3. **禁止结论先行**：辩证分析必须先完成三向推导，再归纳结论，不能倒推
4. **禁止遗漏副作用**：每次给出建议时，必须同步指出潜在副作用或风险
