## 📐 SKILL 文件规范

每个 SKILL 文件应包含以下标准结构：

```
# SKILL 名称
> 一句话功能定位

模块归属 / 状态 / 脚本文件

## 功能定位（能做什么）
## 分析维度（怎么做）
## 在模块中的位置（文件路径关系）
## 输出规范（输出示例）
## 调用接口（代码/命令）
## 执行红线（禁止事项）
## 开发计划（未来 Phase）
```

## ➕ 新增技能流程

1. 在对应模块的 `collectors/` 或根目录新建 `SKILL_技能名.md`
2. 按上方规范结构编写技能文档
3. 更新对应模块的 `README.md` 技能表格
4. 在本文件（`SKILLS_INDEX.md`）添加索引条目
5. 更新 `09_update_log/` 变更记录
