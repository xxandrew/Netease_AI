## B. 方案说明

### [1] 双人动作 · 发起方流程
design: design/01-双人动作-发起.png, design/02-双人动作-右侧窗.png, design/03-双人动作-等待确认.png

> 发起方通过点击场景中对方角色唤出操作菜单，选择双人动作后进入动作选择面板，选定动作后等待对方确认。

#### 1. 底部快捷栏
**类型：** 图标按钮组
1. 显示：
   a. 位于屏幕左下角，横向排列 3 个图标按钮：聊天（气泡图标）/ 动作（双人舞蹈图标）/ 拍照（相机图标）
2. 交互说明：
   a. 点击动作按钮 → 打开单人动作选择面板（见界面[5]）
   b. 其他按钮见对应功能章节
**控件状态：** 常态 / 按下态 / 点击热区(≥80px)

#### 2. 对方角色点击区
**类型：** 场景 UI
1. 显示：
   a. 点击场景中的其他玩家角色后，对方头顶出现操作菜单图标组（双人动作图标 + 随行图标）
   b. 同时显示手指点击引导动效
2. 交互说明：
   a. 点击对方角色 → 头顶弹出操作图标组
   b. 点击双人动作图标 → 右侧动作选择面板展开（界面[1]→[2]）
   c. 点击随行图标 → 发起随行邀请（见界面[3]）
   d. 点击场景其他位置 → 操作图标组收起
**控件状态：** 隐藏态 / 显示态（操作图标组展开）

#### 3. 动作选择面板
**类型：** 列表控件
1. 显示：
   a. 从屏幕右侧滑入，半透明深色背景，左侧为游戏场景
   b. 主区域：3 列网格排列的动作图标卡片，每个卡片为深灰圆角方形底板 + 白色双人动作剪影图标
   c. 右侧为可滚动的单列图标列表（更多动作）
   d. 选中某动作后：对应卡片切换为浅色高亮底板
2. 交互说明：
   a. 点击任意动作卡片 → 卡片高亮，向对方发起动作邀请，进入等待确认态（界面[2]→[3]）
   b. 点击右侧列表图标 → 同上，发起对应动作邀请
   c. 点击场景区域（面板左侧）→ 关闭面板
   **滑动热区：** 右侧列表支持上下滚动
**控件状态：** 常态 / 卡片-选中/等待确认态（高亮）

### [2] 双人动作 · 受邀方流程
design: design/04-双人动作-同意.png

> 受邀方在自己角色头顶收到双人动作邀请图标，点击即同意。

#### 1. 邀请图标（受邀方头顶）
**类型：** 场景 UI
1. 显示：
   a. 位于受邀方自己角色头顶，显示被邀请的双人动作图标 + 手指点击引导动效
   b. 触发条件：对方发起双人动作邀请后显示
   c. 消失条件：点击同意后隐藏；**@策划** 超时未操作是否自动消失？倒计时多长？
2. 交互说明：
   a. 点击头顶图标 → 同意双人动作，双方开始播放动作
**控件状态：** 显示态 / 隐藏态 / 点击热区(≥80px)

### [3] 随行 · 发起与同意
design: design/05-随行-发起.png, design/06-随行-等待同意.png, design/07-随行-待机-发起方.png, design/08-随行-待机-跟随方.png, design/10-随行-受邀方同意.png

> 随行功能入口与双人动作共用同一套「点击对方头顶操作菜单」流程。

#### 1. 对方角色点击区（随行入口）
**类型：** 场景 UI
1. 显示：
   a. 同界面[1]，点击对方角色后头顶出现操作图标组（含随行图标）
2. 交互说明：
   a. 点击随行图标 → 向对方发起随行邀请，进入等待同意态
**控件状态：** 同界面[1]对方角色点击区

#### 2. 随行邀请图标（受邀方头顶）
**类型：** 场景 UI
1. 显示：
   a. 受邀方自己角色头顶出现随行邀请图标 + 手指点击引导动效
   b. 触发条件：对方发起随行邀请后显示
   c. 消失条件：点击同意后隐藏；**@策划** 超时未操作是否自动消失？
2. 交互说明：
   a. 点击头顶图标 → 同意随行，双方进入随行待机态
**控件状态：** 显示态 / 隐藏态 / 点击热区(≥80px)

#### 3. 随行待机态（发起方视角）
**类型：** 纯展示
1. 显示：
   a. 随行成功后，跟随方角色自动跟随发起方移动
   b. 发起方视角：画面中出现两个相同角色并排，无额外 UI 叠加
   c. **@策划** 随行中是否有状态图标显示在 HUD 上（如随行中标识）？

#### 4. 随行待机态（跟随方视角）
**类型：** 纯展示
1. 显示：
   a. 跟随方视角：自己角色（跟随方）在画面中心，发起方角色在场景中
   b. 可点击发起方角色触发操作菜单（含取消随行图标）

### [4] 随行 · 取消方式
design: design/09-随行-发起方取消.png, design/11-随行-大幅摇杆取消.png, design/12-随行-小幅摇杆取消.png

> 随行有三种取消方式，触发条件不同，分别说明。

#### 方式 A · 发起方主动取消
design: design/09-随行-发起方取消.png
1. 显示：
   a. 发起方点击跟随方角色 → 对方头顶出现取消随行图标（双人+×图标）
2. 交互说明：
   a. 点击头顶取消图标 → 取消随行，双方退出随行状态
**控件状态：** 显示态 / 按下态 / 点击热区(≥80px)

#### 方式 B · 跟随方大幅移动摇杆（自动退出）
design: design/11-随行-大幅摇杆取消.png
1. 显示：
   a. 跟随方大幅偏移摇杆时，发起方头顶出现取消随行图标，同时摇杆附近出现取消图标
   b. 无需点击，大幅移动摇杆超出阈值后自动退出随行
2. 交互说明：
   a. 摇杆偏移超过阈值 → 自动取消随行，双方退出随行状态
   **@策划** 「大幅移动」的摇杆偏移阈值是多少？

#### 方式 C · 跟随方小幅移动摇杆（点击取消）
design: design/12-随行-小幅摇杆取消.png
1. 显示：
   a. 跟随方小幅移动摇杆时，发起方头顶出现取消随行图标，摇杆附近无额外图标
   b. 需主动点击头顶取消图标才能退出
2. 交互说明：
   a. 小幅移动摇杆 → 发起方头顶出现取消随行图标
   b. 点击头顶取消图标 → 取消随行
   c. 不点击 / 停止移动摇杆 → 取消图标消失，继续随行
   **@策划** 取消图标显示后若不操作，多久消失？

### [5] 单人动作 · 发起与切换双人动作
design: design/13-单人动作-发起.png, design/14-单人动作-右侧窗.png, design/15-单人切双人动作.png, design/16-单人切双人提示.png

> 单人动作通过底部动作按钮发起，选择双人类动作时，系统提示选择互动对象。

#### 1. 动作按钮（底部快捷栏）
**类型：** 图标按钮
1. 显示：
   a. 位于底部快捷栏中间，双人舞蹈剪影图标
2. 交互说明：
   a. 点击 → 右侧单人动作选择面板展开
**控件状态：** 常态 / 按下态 / 点击热区(≥80px)

#### 2. 单人动作选择面板
**类型：** 列表控件
1. 显示：
   a. 从屏幕右侧滑入，布局与双人动作选择面板相同：3列网格 + 右侧单列滚动列表
   b. 面板内同时包含单人动作和双人动作卡片
2. 交互说明：
   a. 点击单人动作卡片 → 角色立即播放对应单人动作，面板保持展开
   b. 点击双人动作卡片 → 面板顶部出现提示「请选择想要互动的玩家」，引导点击场景中的对方角色
   c. 点击场景区域（面板左侧）→ 关闭面板
   **滑动热区：** 右侧列表支持上下滚动
**控件状态：** 常态 / 卡片-双人动作已选中（等待选玩家态）

#### 3. 选择互动玩家提示
**类型：** 纯展示
1. 显示：
   a. 选中双人动作后，面板顶部出现提示文字「请选择想要互动的玩家」
   b. 触发条件：在单人动作面板中点击双人类动作卡片时显示
   c. 消失条件：点击场景中对方角色 / 取消选择后隐藏
2. 交互说明：
   a. 按提示点击场景中的对方角色 → 向对方发起双人动作邀请，流程同界面[1]/[2]

### [6] 聊天 · 输入与发送
design: design/17-聊天-点击聊天.png, design/19-聊天-键盘无字.png, design/20-聊天-键盘有字.png, design/18-聊天-对话发出.png

> 点击底部聊天按钮唤起键盘输入，发送后消息以气泡形式显示在角色头顶。

#### 1. 聊天按钮（底部快捷栏）
**类型：** 图标按钮
1. 显示：
   a. 位于底部快捷栏最左侧，气泡图标
2. 交互说明：
   a. 点击 → 系统键盘弹出，输入框获得焦点
**控件状态：** 常态 / 按下态 / 点击热区(≥80px)

#### 2. 聊天输入框
**类型：** 输入框
1. 显示：
   a. 键盘弹出后，屏幕下半部分显示系统键盘，键盘上方为横贯全宽的输入框
   b. 无字态：输入框内显示占位符文字「聊天」，光标闪烁
   c. 有字态：输入框内显示已输入文字，占位符消失
   d. 右下角「发送」按钮始终为蓝色可点击态
2. 交互说明：
   a. 输入文字 → 输入框实时显示内容
   b. 点击「发送」→ 消息发送，键盘收起，角色头顶出现对话气泡
   c. 点击键盘以外区域 / 收起键盘 → 取消输入，键盘收起
   **@策划** 输入框是否有字数上限？超出后如何处理？
**控件状态：** 无字态（占位符）/ 有字态 / 发送中

#### 3. 发送按钮
**类型：** 文字按钮
1. 显示：
   a. 位于键盘右下角，蓝色圆角矩形底板 + 白色「发送」文字
2. 交互说明：
   a. 点击 → 发送消息，键盘收起
**控件状态：** 常态（蓝色）/ 按下态 / 点击热区(≥80px)

#### 4. 对话气泡
**类型：** 纯展示
1. 显示：
   a. 消息发送后，发送方角色头顶出现白色圆角矩形气泡，内容为发送的文字
   b. 触发条件：点击「发送」后出现
   c. 消失条件：**@策划** 气泡停留时长？是否自动消失？
2. 交互说明：
   a. 无点击交互，为纯状态展示控件
**控件状态：** 显示态 / 隐藏态
