# 交互文档字段清单（条目级规范）

> 版本：v1.5 | 维护：jinjiang + 喵喵 | 更新：2026-04-13
>
> 用途：明确逐模块撰写时的字段结构、格式、HTML样式对应关系

---

## 一、文档整体结构（v1.5）

v1.5 起交互文档统一采用三段式结构：**文档头 → 需求范围 → 方案说明（按界面分块）**。

```
# 文档标题                          ← 页面 <title>，不在正文渲染

<!-- 文档头 -->
icon: 🌿                           ← 游戏/项目图标（emoji 或图片路径）
version: #A20483                   ← 需求版本号
title: 山丘模组订阅增值              ← 文档标题
planner: 林晓东                     ← 策划姓名
uxd: 喵喵                          ← 设计师姓名

<!-- 需求范围（A 节） -->
## A. 需求范围
- 界面名称1
- 界面名称2
...

<!-- 方案说明（B 节） -->
## B. 方案说明

### [1] 界面名称                    ← 界面块标题，编号用方括号整数
design: design/iface-1.png         ← 界面截图路径（相对 case 目录）

#### 1. 模块名称                   ← 模块层级 = 控件说明
**类型：** xxx
1. 显示：
   ...
2. 交互说明：
   ...
**控件状态：** xxx
**@策划** xxx

#### 2. 模块名称（带子模块）
**类型：** 复合控件

1. 显示：（模块整体说明）
   ...

##### 1 子模块名称                 ← 子模块，数字编号，无方括号
1. 显示/交互：
   ...
**控件状态：** xxx

##### 2 子模块名称
...

### [2] 下一个界面
...

## 需求缺漏                         ← 固定标题
### P0
- **问题标题**
  描述

## 修改记录
| 版本 | 日期 | 修改内容 | 修改人 |
```

> 💡 **v1.5 更新说明**：
> - 新增文档头（`icon` / `version` / `title` / `planner` / `uxd`）
> - 新增需求范围节（`## A. 需求范围`），列出涉及界面缩略图
> - 方案说明按界面分块（`### [N] 界面名称`），左图右文渲染
> - 废弃旧式 `### N.N 子模块` 独立卡片写法，改为 `##### N 子模块名称` 内嵌在父模块卡片中
> - 模块编号仍为整数（`#### 1. 名称`），界面内局部编号

## 二、字段详细规范

### 2.1 文档头字段

文档头写在 MD 文件顶部，每条一行，`key: value` 格式：

| key | 格式 | 示例 | 是否必须 |
|-----|------|------|---------|
| `icon` | emoji 或图片路径 | `🌿` | ✅ |
| `version` | `#需求编号` | `#A20483` | ✅ |
| `title` | 纯文字 | `山丘模组订阅增值` | ✅ |
| `planner` | 姓名 | `林晓东` | ✅ |
| `uxd` | 姓名 | `喵喵` | ✅ |

HTML 渲染为文档头 banner（图标 + 版本 + 标题 + 人员 + 图例色块），**不吸顶**。

---

### 2.2 结构层级

#### 界面块标题
| 属性 | 规范 |
|------|------|
| Markdown格式 | `### [N] 界面名称` |
| 编号规则 | 方括号整数 + 空格 + 名称 |
| 正确示例 | `### [1] 订阅` |
| 错误示例 | `### 1. 订阅`（无方括号）、`### [1.1] 弹窗`（小数） |
| HTML样式 | `[N]` monospace badge + 界面名称 17px 加粗，下边线分隔 |
| 附加字段 | `design: 路径` 指定界面截图，渲染为左侧图片区 |
| 侧边栏 | `[1] 界面名称` 一级条目 |

#### 模块标题（界面内控件）
| 属性 | 规范 |
|------|------|
| Markdown格式 | `#### N. 模块名称` |
| 编号规则 | 数字 + 英文点 + 空格 + 名称，**界面内局部编号（从 1 起）** |
| 正确示例 | `#### 1. 标题栏` |
| 错误示例 | `#### 1 标题栏`（无点）、`## 1. 标题栏`（层级错误） |
| HTML样式 | 橙色圆点 badge + 橙色标题文字，渲染为右侧控件卡片标题 |
| HTML类名 | `.module-header-row` + `.module-num` + `.module-header-title` |
| 侧边栏 | `① 模块名称` 缩进子项，选中时左边线高亮 |

#### 子模块（内嵌在父模块卡片中）
| 属性 | 规范 |
|------|------|
| Markdown格式 | `##### N 子模块名称` |
| 编号规则 | **纯数字**（无点）+ 空格 + 名称，界面内局部编号 |
| 正确示例 | `##### 1 购买按钮（黄色）` |
| 错误示例 | `##### 1. 购买按钮`（带点，会被识别为模块）、`### 4.1 xxx`（旧式，已废弃） |
| HTML样式 | 黄色方角 badge（数字） + 子模块名称，分割线上方 |
| HTML类名 | `.sub-mod-sep` + `.sub-mod-badge` + `.sub-mod-name` |
| 使用场景 | 父模块内有多个独立子控件，需分别列字段时使用 |

#### 简写控件规范（type-only 模块）

当某个控件仅需声明类型（如通用组件、标准导航栏），不展开显示/交互字段时，只写 `**类型：**` 一行即可。

**正确写法：**
```markdown
#### 2. 顶部标题栏
**类型：** 通用组件，页面标题=「金苹果奖·年度评选」
```

**渲染规则：**
- 渲染为**标准 widget-block 卡片**，与有内容的控件样式一致
- 不得使用紧凑横排或特殊灰色样式（由脚本自动处理，无需手动控制）
- 若后续需补充字段，直接在 `**类型：**` 后添加 `1. 显示：` 等字段即可，无需改结构

### 2.3 字段规范

#### 字段1：类型（必须）
| 属性 | 规范 |
|------|------|
| Markdown格式 | `**类型：** 内容` |
| 位置要求 | 必须是控件的**第一个字段** |
| 内容示例 | `按钮`、`复合控件 · Tab导航`、`通用组件，页面标题=活动奖励` |
| HTML样式 | 灰色斜体 13px，位于控件标题下方 |
| HTML类名 | `.widget-type` |
| 特殊用法 | 通用组件只写 `**类型：**` 一行，不展开其他字段；渲染为标准卡片 |

#### 字段2：显示（可选）

| 属性 | 规范 |
|------|------|
| Markdown格式 | `1. 显示：` + 换行 + `   a. xxx`（缩进3格） |
| 编号规则 | 在模块内从 `1.` 起编，与「交互说明」共享同一序号序列 |
| 一级子项 | `a. b. c.` 字母编号，缩进3格 |
| 深层嵌套 | `·` 项目符号，再缩进3格，用于说明子项的补充规则 |
| 元素引用 | `【元素名】` 标注具体元素名称、状态 tag 等专有名词 |
| 图示占位 | 元素数量多（≥5项）或位置关系重要时，末尾单独一行写 `【需要图示】` |
| 图示渲染规则 | `【需要图示】` 是**块级独立元素**，渲染器会自动将其从字段列表内容中提取出来，在列表之后单独输出 `.diagram-hint-row` 容器（深色底板 + 绿色圆点）；**无论写在字段内容的哪一行、缩进多少，均不受列表缩进影响** |
| HTML样式 | 字段标签：灰色14px(`.field-label`)；内容：白色有序列表(`.field-content ol`) |
| 内容样式 | 列表项间距 5px，行高 1.85 |

**正确示例**（基础）：
```markdown
1. 显示：
   a. 当前进度
   b. 进度条
   c. 奖励（物品icon 品级 数量）
```

**正确示例**（带深层嵌套 + 元素引用 + 图示占位）：
```markdown
1. 显示：
   a. 服主头像
   b. 服务器名称
   c. 山头档位 + 存档模式及存档名称
   d. 奖励（物品icon 品级 数量）
      · 部分特殊奖励尺寸稍大展示
      · 提示tag【模组】【头饰】在可领取&已领取时隐藏
   e. 游玩按钮
   f. 故事按钮
   【需要图示】
```

#### 字段3：交互说明（可选）

> 原「交互反馈」字段升级版。涵盖所有与交互相关的说明：操作反馈、触发条件、消失条件等，统一在此字段下按子项逐条列出。

| 属性 | 规范 |
|------|------|
| Markdown格式 | `2. 交互说明：` + 换行 + `   a. xxx`（缩进3格） |
| 兼容旧格式 | `2. 交互反馈：` 仍可被解析器识别，但新文档统一用「交互说明」 |
| 编号规则 | 紧接「显示」段落的序号，通常为 `2.`；无显示段落时为 `1.` |
| 一级子项 | `a. b. c.` 字母编号，缩进3格；多种条件并列时各占一条 |
| 交互反馈写法 | `手势 (+ 【交互目标】) → 交互结果`（箭头连接） |
| 目标引用 | 有多个点击区域时，用 `【元素名】` 明确操作对象 |
| 触发条件写法 | `触发条件：XXXX时/后触发` |
| 消失条件写法 | `消失条件：XXXX时/后消失` |
| **→ 使用限制** | 箭头格式 `→` **只用于「操作行为 → 结果」**，不用于状态描述（❌ `已拥有时 → 显示已拥有`） |
| 子情况格式 | 一个交互有多种结果时，用 `·` 缩进写子情况 |
| 图示占位 | 有多个点击热区时，末尾单独一行写 `【需要交互图示，热区 ≥ 80px】` |
| 图示渲染规则 | 同「显示」字段；文本含「热区」时自动用蓝色圆点（`.diagram-tag-hotzone`） |
| HTML样式 | 同「显示」字段；箭头 `.arrow` 白色加粗 |

**正确示例**（交互反馈）：
```markdown
2. 交互说明：
   a. 点击 → 打开购买确认弹窗
   b. 长按 → 触发其他行为
```

**正确示例**（带目标引用 + 交互图示占位）：
```markdown
2. 交互说明：
   a. 点击【服主头像】→ 跳转个人资料页
   b. 点击【已装载模组数量】→ 打开模组列表弹窗
   c. 点击【游玩按钮】→ 开始游戏
   d. 点击【故事按钮】→ 跳转故事模式
   【需要交互图示，热区 ≥ 80px】
```

**正确示例**（带子情况）：
```markdown
2. 交互说明：
   a. 点击【前往按钮】→
      · 已解锁：跳转至对应界面
      · 未解锁：提示toast<条件不满足，无法前往>
```

**正确示例**（含触发 + 消失条件）：
```markdown
2. 交互说明：
   a. 点击【确认按钮】→ 提交数据，关闭弹窗
   b. 点击蒙层 → 关闭弹窗
   c. 触发条件：点击「?」按钮后触发
   d. 消失条件：点击关闭/确认按钮后消失
```

**正确示例**（纯被动控件，无主动交互）：
```markdown
2. 交互说明：
   a. 触发条件：活动状态变为「已结束」后自动出现
   b. 消失条件：用户点击后消失，下次进入不再显示
```

#### 合并写法：显示 + 交互同行（简单模块）

**适用条件：** 子项 ≤ 4 个，且每项交互逻辑简单（无状态分支）。

**格式：** 在 `1. 显示：` 子项末尾用 `（点击 → 行为）` 内联写交互，不单独写 `2. 交互说明：`

**正确示例：**
```markdown
1. 显示：
   a. 标题
   b. 问号按钮（点击 → 打开通用说明弹窗）
   c. 活动时间
```

**❌ 不适合合并的情况（应拆开写）：**
- 某个元素有多种状态下的不同交互
- 交互结果复杂（需要子情况 `·`）
- 有多个热区需要图示说明（`【需要交互图示】`）

#### 字段4：控件状态（可选）
| 属性 | 规范 |
|------|------|
| Markdown格式 | `**控件状态：** 状态A / 状态B / 状态C` 或 `**控件状态：** [状态A] [状态B]` |
| 分隔方式 | `/` 分隔（推荐）或 `[方括号]` 包裹，效果相同 |
| HTML样式 | `.state-row` 黑色底板圆角容器 + `.state-tag` 灰底胶囊 |
| 胶囊样式 | 背景 `#2a2a2e`，边框 `#3a3a40`，圆角 6px，padding 4px 12px |
| 自动转换 | `/` 分隔的内容会自动切分渲染为多个胶囊 |
| 多控件状态 | 模块含多个控件时，使用 `控件名-状态名` 格式，渲染时分组显示 |
| ⚠️ 禁止写比值/分数 | **状态值中不能包含 `/`**，如 `0/17`、`10/100` 等比值格式会被截断为多个残缺胶囊；此类内容应放进「显示」字段描述，或改写为文字（如「已满」「已达上限」） |

---

#### 字段4.5：子控件（可选）

用于声明当前模块内嵌的子控件类型，供 validate 自动检查子控件必要字段，HTML 以弱显示（小字灰色胶囊）渲染。

| 属性 | 规范 |
|------|------|
| Markdown格式 | `**子控件：** 类型A / 类型B` |
| 位置 | 紧跟 `**类型：**` 之后，在 `1. 显示：` 字段之前 |
| HTML样式 | `.subwidget-row` 弱显示行 + `.subwidget-chip` 灰色小字胶囊（opacity: 0.5） |
| validate行为 | 对每个声明类型运行 WIDGET_TEMPLATE_RULES 检查，验证必要字段（按下态/唤起键盘等）是否在模块内就地描述 |
| 适用场景 | 卡片、复合控件、列表等包含按钮/输入框/开关等子控件的模块 |

**示例：**
```markdown
#### N. 好友列表
**类型：** 列表 / 滚动区
**子控件：** 按钮 / 图标按钮

1. 显示：
   a. 每条卡片：头像、昵称、「加好友」按钮（绿色）、「发消息」图标按钮
2. 交互说明：
   a. 点击「加好友」→ ...

**控件状态：** 加好友按钮-常态 / 加好友按钮-按下态 / 发消息按钮-常态 / 发消息按钮-按下态
**滑动热区：** 整条卡片垂直滑动
```

---

**正确示例**（单控件）：
```markdown
**控件状态：** 常态 / 按下态 / 禁用态
**控件状态：** [常态] [按下态] [禁用态]
```

**v1.2 推荐示例**（多控件状态分组）：
```markdown
**控件状态：** 山头条目-未选中态 / 山头条目-选中态 / 创建按钮-常态 / 创建按钮-按下态
```

**渲染效果**（多控件时分组显示）：
```
控件状态
┌──────────────────────────────────┐
│ 山头条目   未选中态  选中态        │
├──────────────────────────────────┤
│ 创建按钮    常态     按下态        │
└──────────────────────────────────┘
```

**⚠️ 异型按钮附加规则**

没有明确视觉边缘的按钮（图标按钮、纯文字按钮、浮动操作按钮等），**必须在控件状态后追加热区图示**，并说明最小点击热区尺寸（≥ 80px）：

```markdown
**控件状态：** 常态 / 按下态 / 点击热区(≥80px)
```

渲染效果：三个并排胶囊，`点击热区(≥80px)` 自动显示为蓝色调胶囊（区分真实状态与图示说明）。

> 判断依据：用户是否能清楚看出"哪里可以点"。有明确矩形/圆形边框的按钮不需要；图标类、文字链接类、浮动类通常需要。

#### 字段5：滑动热区（可选）
| 属性 | 规范 |
|------|------|
| Markdown格式 | `**滑动热区：** 描述可滑动区域` |
| 使用场景 | 列表、滚动区、可滑动卡片等必须写 |
| HTML样式 | 同「显示」字段，自动追加「热区示意图」灰色标签 |
| 标签样式 | `.hotzone-tag` 黑底、灰边框、斜体 |

**正确示例**：
```markdown
**滑动热区：** 右侧内容区全区域可纵向滑动
```

#### 字段6：空态（可选）
| 属性 | 规范 |
|------|------|
| Markdown格式 | `**空态：** 空态文案或处理说明` |
| 使用场景 | 列表类控件建议写 |
| HTML样式 | 同「显示」字段 |

**正确示例**：
```markdown
**空态：** 显示「暂无数据」插图
```

### 2.4 子模块写法（v1.5，替代旧式内联子控件）

子模块用 `#####` 标题内嵌在父模块 MD 块内，渲染时合并在同一控件卡片中。

| 属性 | 规范 |
|------|------|
| Markdown格式 | `##### N 子模块名称`（五个 `#`，纯数字无点） |
| 字段 | 子模块下可写所有普通字段（显示/交互/控件状态等） |
| HTML样式 | `.sub-mod-sep` 分割线 + `.sub-mod-badge` 黄色方角数字 + `.sub-mod-name` |
| 侧边栏 | 不单独显示，归属父模块条目 |

**正确示例**：
```markdown
#### 2. 套餐价格区

**类型：** 复合控件

1. 显示：
   a. 月价 / 年价 / 买断价
   b. 划线原价（有折扣时显示）

##### 1 折扣标签
1. 有折扣：显示「省 ¥XX」红色气泡
2. 无折扣：隐藏
**控件状态：** 有折扣 / 无折扣（隐藏）

##### 2 立即订阅按钮
1. 未登录 → 提示登录弹窗
2. 已登录 → 调起支付页
**控件状态：** 常态 / 处理中（loading）
```

### 2.5 @职能

|------|------|
| Markdown格式 | `**@职能** 问题描述` |

## 五、图示占位规范

所有图示占位标签共享同一底板样式（`background: #0e0e12; border-radius: 6px`），通过不同颜色圆点区分类型。

| # | 类型 | MD 写法 | 颜色点 | 触发方式 | 位置 |
|---|------|--------|--------|---------|------|
| 1 | 显示说明图示 | `【需要图示】` | 🟢 绿色 `#48C75F` | 手动，在 `1. 显示：` 字段内 | 显示内容末尾 |
| 2 | 交互说明图示 | `【需要交互图示，热区 ≥ 80px】` | 🔵 蓝色 `#4a9eff` | 手动，在 `2. 交互说明：` 字段内 | 交互内容末尾 |
| 3 | 热区图示 | `【需要热区示意图，热区 ≥ 80px】` | 🔵 蓝色 `#4a9eff` | ① 自动：写 `**滑动热区：**` 字段时生成；② 手动：异型按钮在控件状态后追加 | 热区字段末尾 / 控件状态后 |
| 4 | 控件状态图示 | `【需要控件状态图示】` | 🟣 紫色 `#a855f7` | 手动，写在 `**控件状态：**` 字段下一行 | 控件状态后 |
| 5 | 通用组件图示 | `【需要通用组件图示：xxx名称】` | 🟡 黄色 `#F5A623` | 手动，在 `2. 交互说明：` 字段内 | 提到通用组件的交互项后 |

### 类型5 使用说明

当交互说明中提到弹出/跳转某个**通用界面/弹窗/组件**时，需在本模块内附上该组件的图示占位：

```markdown
2. 交互说明：
   a. 点击【道具框】→ 弹出通用道具说明tips
   b. 点击【分享】→ 弹出通用分享面板
   【需要交互图示，热区 ≥ 80px】
   【需要通用组件图示：通用道具说明tips】
   【需要通用组件图示：通用分享面板】
```

**命名规范**：`【需要通用组件图示：具体组件名称】`，组件名称与交互说明中提到的一致。

### 渲染规则

- 所有图示占位均为**块级独立元素**，渲染器自动从字段 content 中提取，不受列表缩进影响
- 类型判断顺序：含「热区」→ 蓝色；含「控件状态图示」→ 紫色；含「通用组件图示/通用界面图示/通用弹窗图示」→ 黄色；其余 → 绿色

## 六、Toast 规范

### 5.1 Toast 文档格式

Toast 内容必须以**尖括号**包围：`<toast内容>`

| 场景 | 正确写法 | 错误写法 |
|------|---------|---------|
| 交互反馈 | `点击 → 提示toast<模组已装载>` | `点击 → toast提示「模组已装载」` |
| 状态变更反馈 | `切换后提示toast<已切换为升序排列>` | `切换后弹toast显示已切换` |

**完整示例**：
```markdown
**交互：**
1. 点击排序按钮 → 提示toast<已切换为升序排列>
2. 点击装载按钮 → 
   a. 装载成功：提示toast<模组已装载>
   b. 装载失败：提示toast<装载失败，请重试>
```

## 六、常见错误对照表

| 错误 | 正确 | 说明 |
|------|------|------|
| `## 1 模块名` | `## 1. 模块名` | 模块标题必须用英文点 |
| `### 1. 控件名` | `### **1. 控件名**` | 控件标题必须加粗（仅子控件用） |
| `**显示：** 内容` | `**显示：**`<br>`1. 内容` | 字段内容必须用编号列表 |
| `**交互：** 无` | （直接不写） | 无交互则省略整个字段 |
| `### P0 · 标题` | `### P0` + `- **标题**` | 缺漏条目不能用 `·` 符号 |
| `**子控件：**` 写在类型前 | `**类型：**` 必须是第一个 | 类型字段必须排第一 |
| `已拥有时 → 显示已拥有` | `已拥有态：显示「已拥有」` | `→` 只用于交互行为与结果连接 |
| `提示toast「内容」` | `提示toast<内容>` | toast必须用尖括号包围 |
| `## 1. 模块` 下无字段 | 添加 `**类型：**` 等字段 | 扁平结构下模块必须有内容 |
| `**控件状态：**`<br>`- 状态A` | `**控件状态：** 状态A / 状态B` | 控件状态不能用 `- ` 列表 |
| `**控件状态：** 有记录 / 空（0/17）` | 移入「显示」字段，或改写为 `有记录 / 空态` | 状态值中含 `/` 的比值（如 `0/17`、`10/100`）会被截断为残缺胶囊 |
