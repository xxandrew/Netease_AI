### 按钮 Button

> 🔍 **描述**：长方形或正方形控件。内容类型分三种：仅文本、图标+文本、仅图标。颜色语义：一级绿色（主操作）、二级浅灰色、三级深灰色（少见）、付费黄色、负向红色、特殊紫色（仅山头系统）。三态：常态 / 按下态 / 禁用态（灰色半透明）；部分按钮有 loading 态或标签（红点/new 等）。

**必须说明：**

1. 交互说明：写清点击后的反馈结果（跳转目标 / toast 内容 / 触发逻辑等）
2. 控件状态：必写常态、按下态；有禁用逻辑时写禁用态；有 loading 态时写加载中；有标签时注明

**可选说明及条件：**

- 有多种触发结果时（如根据当前状态不同），在交互说明中分条描述
- 按钮包含图标时，**需加「显示」字段**，注明「显示 XX 图标 + 文本」；纯文本按钮不写「显示」字段
- 按钮有标签（红点/new 等）时，**需加「显示」字段**，注明标签的显示逻辑

**特殊情况：**

- 付费按钮（黄色）：需注明货币类型及价格显示规则；参见「付费按钮」独立控件条目
- 负向语义按钮（红色）：交互说明需注明是否弹确认弹窗
- 紫色按钮：仅用于「山头」系统，其他界面不应出现

**模板：**

纯文本按钮：

```markdown
#### N. 按钮名称
**类型：** 按钮
1. 交互说明：
   a. 点击 → xxx
**控件状态：** 常态 / 按下态
```

含图标按钮：

```markdown
#### N. 按钮名称
**类型：** 按钮
1. 显示：
   a. 显示 XX 图标 + 文本「xxx」
2. 交互说明：
   a. 点击 → xxx
**控件状态：** 常态 / 按下态
```

**示例：**

备份按钮（含禁用态 + loading 态）：

```markdown
#### 3. 备份按钮
**类型：** 按钮
1. 交互说明：
   a. 可用时：点击 → 按钮变为 loading 状态，完成后 toast 提示「已备份」
   b. 禁用时：点击 → toast 提示「存储空间已满」
**控件状态：** 常态 / 按下态 / 加载中 / 禁用态
```

### 付费按钮 Purchase Button

> 🔍 **描述**：绝大多数为黄色按钮，部分情况也会出现其他颜色，识别时参考颜色 + 按钮上的信息。分三类：普通购买按钮（文本，如「购买」）；钻石购买按钮（钻石图标 + 数值，如「◆ 600」）；抽奖按钮（抽奖券图标 + 数量 + 抽取次数，如「券 10 · 抽1次」）。

**必须说明：**

按类型分别填写：

*普通购买按钮：*
1. 交互说明：写清点击后的反馈（跳转目标 / 执行逻辑）；已购买时点击的反馈
2. 控件状态：常态、按下态、已购买（禁用态）

*钻石购买按钮：*
1. 显示：钻石图标 + 价格数；钻石不足时价格数字变红
2. 交互说明：
   - 钻石充足时的点击反馈
   - 钻石不足时：点击 → 打开通用钻石补足弹窗
   - 已购买时的禁用状态说明
3. 控件状态：常态、按下态、免费、折扣、已购买（禁用态）

*抽奖按钮：*
1. 显示：抽奖券图标 + 数量 + 抽取次数；抽奖券不足时数量变红
2. 交互说明：
   - 抽奖券充足时：点击 → 执行抽奖
   - 抽奖券不足时：点击 → 打开抽奖券补足弹窗（购买数量预填为差额）
3. 控件状态：常态、按下态

**可选说明及条件：**

- 有多种触发结果时（如根据当前状态不同），在交互说明中分条描述
- 按钮包含标签（红点/new 等）时，在「显示」字段注明

**特殊情况：**

- 抽奖按钮有时包含免费单抽，需在控件状态中加「免费单抽（红点）」
- 模组的钻石购买按钮必须包含以下状态：免费 / 原价 / 折扣 / 按下态 / 已购买（禁用态）
  - 免费：去掉钻石数，仅显示文本「免费」
  - 原价：显示钻石图标 + 价格数字
  - 折扣：显示钻石图标 + 折扣价 + 划线原价；有时按钮上附有「X.X折」或「-XX%」标签

**模板：**

普通购买按钮：

```markdown
#### N. 购买按钮名称
**类型：** 付费按钮 · 普通购买
1. 交互说明：
   a. 点击 → xxx
   b. 已购买时：按钮显示为禁用态，文案为「xxx」
**控件状态：** 常态 / 按下态 / 已购买
```

钻石购买按钮：

```markdown
#### N. 购买按钮名称
**类型：** 付费按钮 · 钻石购买
1. 显示：
   a. 显示钻石图标 + 价格；钻石不足时价格数字变红
2. 交互说明：
   a. 钻石充足：点击 → xxx
   b. 钻石不足：点击 → 打开通用钻石补足弹窗
   c. 已购买：按钮置灰，文案为「xxx」
**控件状态：** 常态 / 按下态 / 已购买
```

抽奖按钮：

```markdown
#### N. 抽奖按钮名称
**类型：** 付费按钮 · 抽奖
1. 显示：
   a. 显示抽奖券图标 + 数量 + 抽取次数；抽奖券不足时数量变红
2. 交互说明：
   a. 抽奖券充足：点击 → 执行抽奖
   b. 抽奖券不足：点击 → 打开抽奖券补足弹窗（购买数量预填为差额）
**控件状态：** 常态 / 按下态
```

**示例：**

普通购买按钮 · 订阅按钮：

```markdown
#### 2. 订阅按钮
**类型：** 付费按钮 · 普通购买
1. 交互说明：
   a. 已勾选协议：点击 → 拉起订阅流程（端外）
   b. 未勾选协议：点击 → 弹出同意订阅提示弹窗，同意后拉起订阅流程（端外）
   c. 不可订阅时：按钮显示为禁用态，文案为「已达上限」
**控件状态：** 常态 / 按下态 / 禁用态
```

钻石购买按钮 · 模组购买按钮：

```markdown
#### 3. 购买按钮
**类型：** 付费按钮 · 钻石购买
1. 显示：
   a. 未购买时：显示钻石图标 + 价格；钻石不足时价格数字变红
2. 交互说明：
   a. 钻石充足：点击 → 打开 POS 机弹窗
   b. 钻石不足：点击 → 打开通用钻石补足弹窗
   c. 已购买：按钮置灰，文案为「已拥有」
**控件状态：** 免费 / 原价 / 折扣 / 按下态 / 已拥有
```

抽奖按钮：

```markdown
#### 4. 抽奖按钮
**类型：** 付费按钮 · 抽奖
1. 显示：
   a. 显示抽奖券图标 + 数量 + 抽取次数；抽奖券不足时数量变红
2. 交互说明：
   a. 抽奖券充足：点击 → 执行抽奖
   b. 抽奖券不足：点击 → 打开抽奖券补足弹窗（购买数量预填为差额）
**控件状态：** 免费单抽（红点）/ 常态 / 按下态
```

### 图标按钮 Icon Button

> 🔍 **描述**：仅图标 或 图标+文本 的按钮。图标有时裸露（无边框），有时带边框；文本有时在图标下方，有时在左右。与「按钮 Button」的区别：图标按钮以图标为主体视觉，通常尺寸较小、无色块背景填充。

**必须说明：**

1. 显示：写明显示的图标内容（+ 文本，如有）
2. 交互说明：写清点击后的反馈结果
3. 控件状态：必写常态、按下态；**无边框的图标按钮必须在控件状态中加「点击热区(≥80px)」**

**可选说明及条件：**

- 有标签（红点/new 等）逻辑时，在「显示」字段注明触发条件，并在控件状态中加对应状态
- 按钮有出现/隐藏条件时，在「显示」字段注明（如「仅 XX 视角显示」）

**特殊情况：**

- 部分特殊入口按钮周围包含附属元素（如文字说明、货币图标、数量等），需在「显示」字段一并说明，不可只写图标本体

**模板：**

有边框 / 图标+文本：

```markdown
#### N. 按钮名称
**类型：** 图标按钮
1. 显示：
   a. 显示 XX 图标（+ 文本「xxx」）
2. 交互说明：
   a. 点击 → xxx
**控件状态：** 常态 / 按下态
```

无边框纯图标（需热区标注）：

```markdown
#### N. 按钮名称
**类型：** 图标按钮
1. 显示：
   a. 显示 XX 图标
2. 交互说明：
   a. 点击 → xxx
**控件状态：** 常态 / 按下态 / 点击热区(≥80px)
```

**示例：**

申请按钮（无边框图标 + 下方文本，含红点 + 显示条件）：

```markdown
#### 3. 申请按钮
**类型：** 图标按钮
1. 显示：
   a. 显示申请图标 + 文本「申请」
   b. 仅队长视角显示，队员不显示此按钮
   c. 有新申请时，按钮显示红点
2. 交互说明：
   a. 点击 → 打开入队申请弹窗
**控件状态：** 常态 / 按下态 / 红点 / 点击热区(≥80px)
```

### 页签 Tab

> 🔍 **描述**：横向或纵向排列的切换控件。纵向页签通常位于界面左侧（一级），横向页签通常位于界面偏上方（二级）；部分界面纵向页签也包含嵌套二级。内容大多为纯文本，有时为图标+文本。页签根据页内状态显示红点（逐级向上冒泡）；部分页签有锁定状态，显示锁图标。「页签」与「Tab」含义相同，写法可互换。

**必须说明：**

1. 交互说明：点击 → 切换
2. 控件状态：必写常态、按下态、选中态

**可选说明及条件：**

- 页签包含图标时，**需加「显示」字段**，注明图标+文本
- 有锁定逻辑时，在「显示」字段注明未解锁时额外显示锁图标，控件状态中加「未解锁」
- 有红点逻辑时，在「显示」字段注明触发条件，控件状态中加「红点」
- 有默认选中逻辑时（如进入界面自动选中某Tab），在交互说明中注明

**特殊情况：**

- 底部一级页签：当前游戏内仅「商城」界面使用，其他界面不应出现
- 右侧特殊页签：商城及背包外观界面中，右侧存在拆件页签（套装/头饰/上衣/…），遇到时按实际内容描述

**模板：**

纯文本页签：

```markdown
#### N. xxx 页签
**类型：** 页签
1. 交互说明：
   a. 点击 → 切换
**控件状态：** 常态 / 按下态 / 选中态
```

含锁定 + 红点页签：

```markdown
#### N. xxx 页签
**类型：** 页签
1. 显示：
   a. 未解锁时，额外显示锁图标
   b. 页内有 xxx 时，页签显示红点
2. 交互说明：
   a. 点击 → 切换
**控件状态：** 常态 / 按下态 / 选中态 / 未解锁 / 红点
```

**示例：**

阶段 Tab（含锁定 + 红点 + 默认选中逻辑）：

```markdown
#### 2. 阶段 Tab
**类型：** 页签
1. 显示：
   a. 未解锁时，额外显示锁图标
   b. 有可领取奖励时，页签显示红点
2. 交互说明：
   a. 点击 → 切换下方显示内容（乐魂图、补水及奖励相关）
   b. 进入界面时，自动选中当前最高阶段
**控件状态：** 常态 / 按下态 / 选中态 / 未解锁 / 红点
```

### 开关 Toggle

> 🔍 **描述**：常规开关样式，开/关两侧各有「开」「关」文字标识，当前激活侧为绿色高亮。另有特殊像素风格开关（中间含图标），样式略有不同但逻辑相同，遇到时需注明「特殊开关」。

**必须说明：**

1. 交互说明：点击 → 切换开启/关闭状态
2. 控件状态：必写开启、关闭、按下态

**可选说明及条件：**

- 有禁用态时，控件状态中加「禁用-开」「禁用-关」（如前置条件不满足、前置开关未开启等）
- 开关嵌入设置项中时（设置名称 + 描述 + 开关 同框），**不要单独描述开关**，整个设置项作为一个控件统一说明显示、交互、状态

**特殊情况：**

- 部分系统使用特殊像素风格开关（中间含图标，见图2），遇到时在「类型」字段注明「特殊开关」

**模板：**

单独开关（简单情况）：

```markdown
#### N. 开关名称
**类型：** 开关
1. 交互说明：
   a. 点击 → 切换开启/关闭状态
**控件状态：** 开启 / 关闭 / 按下态
```

设置项（开关嵌入整体条目中）：

```markdown
#### N. xxx 设置项
**类型：** 开关
1. 显示：
   a. 设置名称：xxx
   b. 设置描述：xxx（多行时纵向扩展）
   c. 包含开关
2. 交互说明：
   a. 点击【开关】→ 切换开启/关闭状态
   b. 禁用时：整个设置项置灰，开关不可操作
**控件状态：** 设置项-可用 / 设置项-禁用 / 开关-开启 / 开关-关闭 / 开关-按下态 / 开关-禁用-开 / 开关-禁用-关
```

**示例：**

通用开关设置项（含禁用态）：

```markdown
#### 3. 消息通知设置项
**类型：** 开关
1. 显示：
   a. 设置名称：接收消息通知
   b. 设置描述：开启后将接收来自系统的消息推送（多行时纵向扩展）
   c. 包含开关
2. 交互说明：
   a. 点击【开关】→ 切换开启/关闭状态
   b. 未登录时：整个设置项置灰，开关不可操作
**控件状态：** 设置项-可用 / 设置项-禁用 / 开关-开启 / 开关-关闭 / 开关-按下态 / 开关-禁用-开 / 开关-禁用-关
```

### 单选框 Radio

> 🔍 **描述**：菱形图标+文本的选择控件（本游戏像素风格，无法使用常规圆形，改为菱形）。绿色填充菱形=已选中，空心菱形=未选中。通常以组的形式出现（多个选项并排）。

**必须说明：**

1. 显示：列出选项（选项较多时可简写，如「【全部】【已下载】…共 N 项」）
2. 交互说明：点击 → 切换选中项，并注明热区包含图标+文本
3. 控件状态：必写常态、按下态、选中态、点击热区(≥80px)（热区须覆盖图标+文本整体，不可只标图标）

**可选说明及条件：**

- 有禁用态时，在控件状态中加「禁用」，并在显示或交互说明中注明触发条件

**特殊情况：**

*（暂无，后续补充）*

**模板：**

```markdown
#### N. xxx 单选框
**类型：** 单选框
1. 显示：
   a. 选项：【xxx】【xxx】
2. 交互说明：
   a. 点击 → 切换选中项；热区包含图标+文本
**控件状态：** 常态 / 按下态 / 选中态 / 点击热区(≥80px)
```

**示例：**

下载状态筛选（含显示条件）：

```markdown
#### 3. 下载状态单选框
**类型：** 单选框
1. 显示：
   a. 选项：【全部】【已下载】
   b. 畅玩页签下隐藏此控件
2. 交互说明：
   a. 点击 → 切换选中项；热区包含图标+文本
**控件状态：** 常态 / 按下态 / 选中态 / 点击热区(≥80px)
```

### 奖励道具框 Reward Item

> 🔍 **描述**：方形底板 + 奖励 icon + 数量（数量有时不显示）。底板颜色对应奖励品级：一般-绿 / 稀有-蓝 / 史诗-紫 / 传说-橙 / 神话-彩。点击道具框默认显示通用道具说明 tips；部分场景同时承载领取功能，此时按当前状态执行不同交互。
>
> 🔍 **识别标准**：同时满足以下三点即判定为奖励道具框，**无论 icon 外观是否为常规形状**（活动专属图标如水滴、魂值等同样适用）：
> 1. 有奖励 icon + 背景底板（方形底框）
> 2. 有数量文字，固定显示在右下角，白色文字
> 3. 用于表示玩家可获得的奖励（道具、货币、活动专属资源均算）

> **写法说明**：奖励道具框通常不需单独列为控件节点，在所在模块内就近说明即可：

### 虚拟摇杆 Joystick

> 🔍 **描述**：玩家用拇指拖动控制角色移动方向的圆盘控件。位于屏幕左下区域。分两层：底座圆盘（固定或可浮动）+ 中心滑块（跟随手指偏移）。待机时可收起为小圆点以减少遮挡；激活时展开为完整圆盘。

**必须说明：**

1. 显示：说明待机态（收起/展开）和激活态的外观差异
2. 交互说明：按下/拖动/松开三个阶段的行为
3. 滑动热区：摇杆可响应的屏幕区域（通常为左侧约 1/3 屏幕宽）
4. 控件状态：待机态 / 激活态

**可选说明及条件：**

- 摇杆支持「浮动出现」（手指按下任意位置，摇杆以该点为中心生成）时，在交互说明中注明
- 摇杆有锁定方向（如只能上下/左右）时，在交互说明中注明

**模板：**

```markdown

1. 显示：
   a. 待机态：收起为小圆点，低透明度
   b. 激活态：展开为大圆盘底座 + 中心滑块，滑块跟随手指偏移
2. 交互说明：
   a. 按下并拖动 → 控制角色移动方向，滑块随手指偏移
   b. 松开 → 滑块回中，角色停止移动
   **滑动热区：** 屏幕左侧约 1/3 区域

**控件状态：** 待机态（小圆点）/ 激活态（大圆盘展开）
```

### 技能按钮 Skill Button

> 🔍 **描述**：游戏 HUD 右侧的圆形技能操作按钮，显示技能图标，支持点击/长按两种交互。核心特征：① 图标随角色状态动态切换（如跳跃 → 二段跳图标）；② 有冷却状态（灰色遮罩 + 倒计时）；③ 长按可进入瞄准模式（部分技能）；④ 多个技能按钮组成「技能按钮区」，数量可随场景动态变化。

**必须说明：**

1. 显示：说明图标内容；如有动态图标切换，说明切换条件
2. 交互说明：点击行为；如支持长按，单独说明长按行为
3. 控件状态：必写常态 / 按下态 / 冷却中；有长按行为写长按激活态；有图标切换写对应状态名

**可选说明及条件：**

- 技能按钮区整体有出现/消失逻辑（如长按期间其他按钮隐藏）时，在父容器「技能按钮区」中统一说明，不在单个按钮中重复
- 按钮数量动态变化（如特定场景下新增第 N 个按钮）时，在显示字段注明触发条件，并用 `**@策划**` 确认

**特殊情况：**

- **长按瞄准模式**：长按触发后其他按钮全部隐藏，仅保留当前按钮 + 取消按钮；此场景下被长按的按钮无需单独写「按下态」（长按本身即激活态）
- **二段跳图标切换**：跳跃按钮在空中时图标切换为专属图标，需在显示字段说明切换条件，控件状态分「地面态」和「空中态」列出

**模板：**

普通技能按钮（仅点击）：

```markdown
#### N. XXX 技能按钮
**类型：** 图标按钮

1. 显示：
   a. 显示 XX 图标
2. 交互说明：
   a. 点击 → 释放 XX 技能

**控件状态：** 常态 / 按下态 / 冷却中 / 点击热区(≥80px)
```

支持长按瞄准的技能按钮：

```markdown
#### N. XXX 技能按钮
**类型：** 图标按钮

1. 显示：
   a. 显示 XX 图标
2. 交互说明：
   a. 点击 → 立即释放 XX 技能
   b. 长按 → 进入瞄准模式（其他技能按钮隐藏，场景出现落点指示器）
   c. 长按后松开 → 在当前落点释放技能，退出瞄准模式

**控件状态：** 常态 / 按下态 / 冷却中 / 长按瞄准态 / 点击热区(≥80px)
```

图标动态切换的技能按钮（如跳跃/二段跳）：

```markdown
#### N. 跳跃技能按钮
**类型：** 图标按钮

1. 显示：
   a. 地面时：显示跑跳人物图标
   b. 空中时（已跳起）：图标切换为旋转/二段跳专属图标
2. 交互说明：
   a. 地面时点击 → 角色起跳
   b. 空中时点击 → 触发二段跳，消耗二段跳次数

**控件状态：** 地面-常态 / 空中-二段跳可用 / 空中-二段跳已用尽 / 按下态 / 冷却中 / 点击热区(≥80px)
```

### Boss 血条/弱点环 Boss Health Bar

> 🔍 **描述**：附着在 Boss 身体上的血量/弱点耐久度指示器，通常为圆形进度环或线性进度条，随受击实时更新。无玩家点击交互，为纯状态展示控件。

**必须说明：**

1. 显示：形状（圆环/线形）、颜色、附着位置 + 触发/消失条件

### 长按取消按钮 Cancel Button（技能瞄准模式）

> 🔍 **描述**：技能长按瞄准模式下右上角出现的取消按钮，红色圆形背景 + × 图标。随进入/退出瞄准模式动态显示/隐藏，生命周期与瞄准模式绑定。

**必须说明：**

1. 显示：外观 + 触发条件（进入瞄准模式）+ 消失条件（松开技能按钮后）
2. 交互说明：点击 → 取消技能释放，退出瞄准模式
3. 控件状态：显示态 / 按下态（无「隐藏态」——隐藏时控件本身不存在）

**模板：**

```markdown
#### N. 取消按钮
**类型：** 图标按钮

1. 显示：
   a. 位于屏幕右上角，红色圆形背景 + 白色 × 图标
   b. 触发条件：进入技能长按瞄准模式时显示
   c. 消失条件：松开技能按钮（无论释放或取消）后隐藏
2. 交互说明：
   a. 点击 → 取消技能释放，退出瞄准模式

**控件状态：** 显示态 / 按下态 / 点击热区(≥80px)
```
